What is an Algorithm?

An algorithm is a list of step-by-step instructions that a computer program follows to solve a problem.

Think of an algorithm as a recipe in cooking: you follow specific steps in a specific order to create a dish. In programming, algorithms guide the computer on how to process data and make decisions.

Key Aspects of an Algorithm

For a procedure to be considered a valid algorithm, it must satisfy three essential properties:

Well-Ordered

The steps must be arranged in a clear, logical sequence. Each step follows the previous one in a precise order.

Example:
Step 1 → Step 2 → Step 3 (not Step 2 → Step 1 → Step 3)

Unambiguous

Each instruction must be clear and precise. There should be no confusion about what each step means or how to execute it.

Good:
"Add 5 to the number"
Bad:
"Add some amount to the number"

Effectively Computable

Each step must be achievable and executable in a finite amount of time. The algorithm must eventually terminate and produce a result, it cannot run forever.

Example:
Calculate 2 + 3 ✓ (computable)
Find the last digit of pi ✗ (infinite)

Simple Algorithm Example

Let's look at an everyday algorithm: Morning Routine. This is a sequence of steps you follow every morning to get ready for work or school.

Morning Routine Algorithm
  1. START
  2. Get out of bed
  3. Take off pajamas
  4. Take a shower
  5. Get dressed
  6. Eat breakfast
  7. Carpool to work
  8. END

Notice how this algorithm is well-ordered (you don't get dressed before take a shower), unambiguous (each step is clear), and effectively computable (each step can be completed in finite time).

Algorithm Representation

Algorithms can be represented in different ways to make them easier to understand and communicate. The two most common methods are Pseudocode and Flowcharts.

1. Pseudocode

Pseudocode is a simplified way of writing an algorithm using plain language that resembles programming code but is easier to read. It uses a mix of natural language and programming-like syntax to describe the logic.

Why use Pseudocode?

Pseudocode allows you to focus on the logic of the algorithm without worrying about the syntax rules of a specific programming language. It's a planning tool before writing actual code.

Pseudocode Example: Calculate Sum of Two Numbers

START
    INPUT number1
    INPUT number2
    sum = number1 + number2
    OUTPUT sum
END
Explanation:
  • START and END mark the beginning and end of the algorithm
  • INPUT means we're receiving values from the user
  • sum = number1 + number2 performs the calculation
  • OUTPUT displays the result to the user

2. Flowchart

Flowchart is a visual representation of an algorithm using shapes and arrows. Each shape represents a specific type of action or decision, and arrows show the flow of control from one step to the next.

Why use Flowcharts?

Flowcharts make it easy to see the overall structure and flow of an algorithm at a glance. They're especially useful for understanding decision points and loops visually.

Flowchart Example: Calculate Sum of Two Numbers

The same algorithm shown as a flowchart uses different shapes connected by arrows to represent each step.

(Visual representation: Start → Input number1 → Input number2 → Process: sum = number1 + number2 → Output sum → End)

Each step in the pseudocode above corresponds to a shape in the flowchart, connected by directional arrows showing the sequence of execution.

Tips for Writing Pseudocode

Follow these guidelines to write clear and effective pseudocode:

1
Write Only One Statement Per Line

Each line should contain only one instruction. This makes the pseudocode easier to read and follow.

✓ Good:
INPUT age
IF age >= 18 THEN
    OUTPUT "You are an adult"
END IF
✗ Bad:
INPUT age IF age >= 18 THEN OUTPUT "You are an adult" END IF
2
Capitalize the Initial Keyword

Write keywords (like IF, THEN, ELSE, FOR, WHILE, INPUT, OUTPUT) in uppercase to distinguish them from variable names.

INPUT username
IF username == "admin" THEN
    OUTPUT "Welcome, Administrator"
ELSE
    OUTPUT "Welcome, User"
END IF
3
Use Indentation to Show Dependency

Indent statements that are inside loops, conditions, or functions. This visually shows which statements belong together.

FOR i = 1 TO 5 DO
    OUTPUT i
    total = total + i
END FOR

The indented lines (OUTPUT i and total = total + i) are executed as part of the FOR loop.

4
End Multiline Structures

Always close multiline structures (IF, FOR, WHILE) with an END statement to clearly mark where they finish.

WHILE count < 10 DO
    OUTPUT count
    count = count + 1
END WHILE

IF score > 50 THEN
    OUTPUT "Pass"
ELSE
    OUTPUT "Fail"
END IF

Understanding Flowcharts

A flowchart is a diagram that uses standardized symbols to represent different types of actions or steps in a process. Arrows connect these symbols to show the flow of execution.

Software for Creating Flowcharts

There are many tools available for drawing flowcharts:

RAPTOR

Educational tool designed specifically for learning algorithms

Microsoft Visio

Professional diagramming software with flowchart templates

Draw.io

Free online flowchart tool, easy to use

Commonly Used Flowchart Symbols

Each symbol in a flowchart has a specific meaning:

Oval
Start / End

Marks the beginning or ending point of the algorithm

→
Arrow / Flowline

Shows the direction of flow from one step to the next

Parallelogram
Input / Output

Represents data entering or leaving the system (INPUT, OUTPUT, PRINT, READ)

Rectangle
Process

Represents a processing step, calculation, or assignment (e.g., sum = a + b)

Diamond
Decision

Represents a yes/no question or condition (IF statements, loops)

What Are Pseudocode and Flowcharts For?

Both pseudocode and flowcharts serve important purposes in algorithm design and communication:

Visualization of Logic

Flowcharts provide a visual roadmap of the algorithm, making it easy to see the overall structure and flow at a glance. Pseudocode provides a textual blueprint.

Ease Understanding

Both methods break down complex problems into simple, manageable steps. This makes it easier for anyone to understand the solution.

Readability and Clarity

They use simple, clear language and symbols that are universally understood, making algorithms accessible to both programmers and non-programmers.

Communication Tool

They serve as a common language for discussing solutions with team members before writing any code.

Planning and Design

Writing pseudocode or drawing flowcharts helps you plan and organize your thoughts before coding. This reduces errors and saves time.

Debugging and Testing

It's easier to spot logical errors in pseudocode or flowcharts than in complex code. You can verify the algorithm's correctness before implementation.

Documentation

They serve as excellent documentation for programs, helping others (or your future self) understand how the program works.

Key Takeaways

  • An algorithm is a step-by-step procedure to solve a problem or complete a task
  • Algorithms must be well-ordered, unambiguous, and effectively computable
  • Pseudocode uses plain language with programming-like structure to describe algorithms
  • Flowcharts use standardized symbols (oval, rectangle, diamond, parallelogram, arrows) to visualize algorithms
  • Write one statement per line in pseudocode and capitalize keywords like IF, WHILE, FOR
  • Use indentation to show which statements belong inside loops or conditions
  • Tools like RAPTOR, Draw.io, and Microsoft Visio can create professional flowcharts
  • Both pseudocode and flowcharts help with planning, communication, debugging, and documentation
