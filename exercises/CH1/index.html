Introduction to C Language

Understand the foundations of programming and the C language environment

8 Topics 10 Exercises ~20 min

What is a Programming Language?

A programming language is a language used by a person to express a process by which a computer can solve a problem.

Types of Programming Language

Programming languages are generally divided into three categories:

1. Machine Language

The lowest-level language, written in binary code (0s and 1s). It is the only language directly understood by the computer's CPU. Extremely fast but difficult for humans to read or write.

Example:
01001000 01100101 01101100
2. Assembly Language

A low-level language that uses symbolic names instead of binary codes. Easier than machine language but but cannot be understand by computer. Requires a translator called an assembler.

Example:
add 2, 3, result;
3. High-Level Language

Closest to human language. Easy to read, write, and maintain. Requires a translator (compiler or interpreter). Widely used today.

Examples:
circle_area = 2 * 3.1415 * 5;

Types of Translators

Since computers only understand machine language (binary), a translator is needed to convert human-readable code into machine code. There are three main types:

Compiler

Translates the entire source code into machine code at once before execution. Read the program at-a-time. Faster than other translator

Interpreter

Translates and executes the source code line by line. Slower than a compiler but easier to debug.

Assembler

Translates assembly language into machine code. Only used for programs written in assembly language.

Other Popular Programming Languages

Besides C, there are many widely-used programming languages today, each with its own strengths and use cases:

Python

General purpose, AI & data science

Java

Enterprise apps, Android development

C++

Game development, system software

JavaScript

Web development, front-end & back-end

Swift

iOS & macOS app development

Kotlin

Modern Android development

Typical C Program Development Environment

Before a C program can run, it goes through several stages. Each stage transforms the code into something the computer can finally execute:

1
Editor The programmer writes the source code using a text editor or IDE (e.g., VS Code). The file is saved and stores on disk.
2
Preprocessor Processes directives that begin with # (e.g., #include, #define). It inserts header files and expands macros before compilation.
3
Compiler Translates the preprocessed source code into object code (machine-readable but not yet executable). Checks for syntax errors during this stage.
4
Linker Combines the object code with library files and other object files to create a complete executable program (.exe).
5
Loader Loads the executable program into the computer's memory (RAM) so it is ready to run.
Execute / CPU The CPU executes the program instructions one by one. The program is now running!

Problems at Execution Time

Even after a program successfully compiles, errors can still occur when the program is running. These are called runtime errors or execution-time errors.

Fatal Errors

Cause the program to terminate immediately. The program cannot continue and crashes. Examples include division by zero, accessing invalid memory.

Example:
int x = 5 / 0;  // Division by zero → program crashes
Nonfatal Errors

Allow the program to continue running but produce incorrect or unexpected results. These are harder to detect because the program does not crash and shows what's wrong.

Example:
int age = -5;  // Program runs, but result is wrong

Your First C Program

Here is the classic "Hello, World!" program, the first program most programmers write in any language. Let's break it down line by line:

#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    return 0;
}
#include <stdio.h> Includes the Standard Input/Output library. Required to use printf() and scanf(). Basically we call the library that contains the function printf() and many others.
int main() The main function, every C program must have one. Execution always starts here.
printf("...\n") Prints text to the screen. \n moves the cursor to a new line (newline character).
return 0; Returns the value 0 to the operating system, indicating the program ended successfully.
{  } Curly braces mark the beginning and end of a block of code (the body of the function). Can also be called domain of the function

Program Development Life Cycle (PDLC)

The PDLC is a structured process used to develop software. It ensures that programs are built correctly and efficiently through defined stages:

1
Analysis

Understand and define the problem clearly. Identify inputs, outputs, and constraints. Answer the question: "What needs to be solved?"

2
Design

Plan the solution before coding. Create flowcharts or pseudocode to outline the logic. Answer the question: "How will the problem be solved?"

3
Implementation / Coding

Write the actual program code in a programming language (in this case, C) based on the design we've created in the previous step.

4
Testing / Debugging

Run the program with various inputs to verify it works correctly. Identify and fix errors (bugs). Debugging is the process of finding and removing those errors.

5
Maintenance

After the program is deployed, update and improve it over time. Fix newly discovered bugs, add features, or adapt the program to new requirements.

Types of Errors in Programming

When writing programs, errors (also called bugs) can occur at different stages. Understanding the type of error helps you fix it faster:

Syntax Error Caught at Compile Time

Violations of the programming language grammar rules. The compiler detects these and the program will not compile until fixed.

Example:
printf("Hello")  // Missing semicolon → syntax error
Runtime Error Occurs During Execution

Errors that occur while the program is running. The program compiles successfully but crashes or behaves unexpectedly during execution.

Example:
int x = 10 / 0;  // Division by zero → program crashes
Logical Error Hardest to Detect

The program compiles and runs without crashing, but produces wrong results. The logic of the program is flawed. These require careful tracing to find.

Example:
average = a + b / 2;  // Should be (a + b) / 2 → wrong answer

Key Takeaways

  • Programming languages come in three levels: machine, assembly, and high-level
  • Compilers translate whole programs at once; interpreters go line by line
  • A C program goes through: Editor → Preprocessor → Compiler → Linker → Loader → CPU
  • Fatal errors crash the program; nonfatal errors produce wrong results silently
  • Every C program must have a main() function, execution starts there
  • The PDLC has 5 stages: Analysis, Design, Implementation, Testing, and Maintenance
  • Syntax errors are caught at compile time; runtime and logical errors appear during execution

What is a Programming Language?

A programming language is a language used by a person to express a process by which a computer can solve a problem. It provides a set of rules and syntax that allows humans to communicate instructions to a computer in a structured and understandable way.

Types of Programming Language
