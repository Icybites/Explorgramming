Modular Programming Concepts

What is Modular Programming?

Modular programming is a software design technique that separates a program into independent, interchangeable modules or functions. Each module performs a specific task and can be developed, tested, and debugged independently.

Divide and Conquer Approach

The "divide and conquer" strategy breaks a large, complex problem into smaller, manageable sub-problems. Each sub-problem is solved by a separate function, making the overall solution easier to develop and maintain.

1
Divide

Break the problem into smaller sub-problems

2
Conquer

Solve each sub-problem independently with functions

3
Combine

Integrate all solutions to solve the original problem

Benefits of Modular Programming

Code Reusability

Write once, use multiple times. Functions can be called from anywhere in the program.

Improved Readability

Smaller, focused functions are easier to read and understand than one large block of code.

Easier Debugging

Isolate and fix errors in specific functions without affecting the entire program.

Team Collaboration

Different team members can work on different functions simultaneously.

Easy Maintenance

Update or modify individual functions without rewriting the entire program.

Built-in Functions (Library Functions)

Built-in functions (also called library functions) are pre-written functions provided by C's standard libraries. These functions are already tested, optimized, and ready to use by including the appropriate header file.

Why Use Library Functions?

Library functions save time and effort. Instead of writing code from scratch, you can use reliable, efficient functions that have been tested by millions of programmers worldwide.

Commonly Used Header Files

#include <stdio.h> Input/Output

Purpose: Provides functions for standard input and output operations.

Common Functions:
  • printf() - Display output
  • scanf() - Read input
  • getchar() - Read a character
  • putchar() - Print a character
Example: 
printf("Hello, World!\n");
scanf("%d", &num);
#include <math.h> Mathematics

Purpose: Provides mathematical functions for complex calculations.

Common Functions:
  • sqrt() - Square root
  • pow() - Power (xy)
  • ceil() - Round up
  • floor() - Round down
  • sin(), cos(), tan() - Trigonometry
Example: 
double result = sqrt(25);    // 5.0
double power = pow(2, 3);    // 8.0
#include <stdlib.h> General Utilities

Purpose: Provides utility functions for memory allocation, conversion, and random numbers.

Common Functions:
  • malloc() - Allocate memory
  • rand() - Generate random number
  • exit() - Terminate program
Example: 
int randomNum = rand() % 100;  // 0-99
int num = atoi("123");         // 123
#include <time.h> Date & Time

Purpose: Provides functions for manipulating and formatting time and date.

Common Functions:
  • time() - Get current time
  • clock() - Get processor time
  • difftime() - Time difference
Example: 
time_t now = time(NULL);
srand(time(NULL));  // Seed random
Part 2

User-Defined Functions

What are User-Defined Functions?

User-defined functions are functions created by the programmer to perform specific tasks. Unlike built-in functions, you design and implement these functions yourself based on your program's needs.

Why Create Your Own Functions?

When library functions don't meet your specific requirements, you create custom functions. This allows you to organize code logically and create reusable components tailored to your application.

Function Definition vs Function Call

Function Definition

The function definition is where you write the actual code that performs the task. It specifies what the function does.

// Function Definition
int add(int a, int b) { 
    int sum = a + b;
    return sum;
}

Function Call

The function call is where you execute the function. It invokes the function to perform its task.

// Function Call
int main() {
    int result = add(5, 3);
    printf("%d", result);
    return 0;
}

Function Header (Signature)

The function header defines the function's interface. It tells the compiler and other programmers how to use the function.

return-value-type function-name(parameter-list) { 
    statements;
}
1. Return Value Type

Specifies the data type of the value the function returns. Common types: int, float, double, char, or void (no return).

2. Function Name

The identifier used to call the function. Should be follow C naming conventions (lowercase, underscores).

calculateSum, findMax, isEven
3. Parameter List

Variables that receive values when the function is called (inputs). Can be empty if no input is needed.

(int a, int b, float c)
4. Function Body

The block of code between { } that contains the statements to be executed. This is where the function's logic resides.

{ statements; return value; }

Complete Example:

#include <stdio.h>

// Function Header: int multiply(int x, int y)
int multiply(int x, int y) {    // Return type: int
                                // Function name: multiply
                                // Parameters: int x, int y
    int result = x * y;         // Function body (statements)
    return result;              // Return statement
}

int main() {    // Function call
    int answer = multiply(5, 4); 
    printf("5 x 4 = %d\n", answer);
    return 0;
}

// Output: 5 x 4 = 20

Void Functions vs Non-Void Functions

Functions can be classified based on whether they return a value:

Non-Void Functions

Functions that return a value after execution. The return type specifies what type of data is returned.

// Non-void function (returns int)
int square(int num) {
    return num * num;
}

int main() {
    int result = square(5);  // result = 25
    printf("Square: %d\n", result);
    return 0;
}
Key Points:
  • Must have a return statement
  • Returns a value to the caller
  • The returned value can be stored, used in expressions, or printed
Void Functions

Functions that do not return a value. They perform an action but don't give back any result.

// Void function (returns nothing)
void printMessage(char name[]) {
    printf("Hello, %s!\n", name);
    // No return statement needed
}

int main() {
    printMessage("Alice");  // Output: Hello, Alice!
    printMessage("Bob");    // Output: Hello, Bob!
    return 0;
}
Key Points:
  • Does NOT need a return statement (or use return; without a value)
  • Performs an action (print, modify variables, etc.)
  • Cannot be used in expressions or assignments

Function Prototype and Multiple Function Calls

A function prototype (also called function declaration) tells the compiler about a function's existence before it's actually defined. This allows functions to be defined in any order and called from anywhere.

Syntax:
return-type function-name(parameter-types);

Example: Functions Calling Other Functions

#include <stdio.h>

// Function Prototypes (Declarations)
int add(int a, int b);
int multiply(int a, int b);
int calculate(int x, int y);

// Main function
int main() {
    int result = calculate(5, 3);
    printf("Final Result: %d\n", result);
    return 0;
}

// Function that calls other functions
int calculate(int x, int y) {
    int sum = add(x, y);          // Call add function
    int product = multiply(x, y); // Call multiply function
    return sum + product;         // Return combined result
}

// Function definitions
int add(int a, int b) {
    return a + b;  // Returns 8
}

int multiply(int a, int b) {
    return a * b;  // Returns 15
}

// Output: Final Result: 23
// Explanation: (5+3) + (5*3) = 8 + 15 = 23

The Return Statement

The return statement serves two purposes:

  • Send a value back to the function caller
  • Exit the function immediately (any code after return is not executed)
int findMax(int a, int b) {
    if (a > b) {
        return a;  // Exit immediately, return a
    } else {
        return b;  // Exit immediately, return b
    }
    // Code here would never execute
}

Passing Values to Functions

When you call a function, you can pass data to it through arguments. The function receives these values through its parameters.

Parameters (Formal Parameters)

Variables listed in the function definition that receive values.

int add(int a, int b) {  // a and b are parameters
    return a + b;
}

Arguments (Actual Parameters)

The actual values passed to the function when it's called.

int result = add(5, 3);  // 5 and 3 are arguments

Pass by Value

In C, arguments are passed by value. This means the function receives a copy of the data, not the original. Changes made to parameters inside the function do NOT affect the original variables.

#include <stdio.h>

void modifyValue(int num) {
    num = num + 10;  // Modifies the COPY, not the original
    printf("Inside function: %d\n", num);
}

int main() {
    int x = 5;
    
    printf("Before function call: %d\n", x);  // 5
    modifyValue(x);                           // Inside function: 15
    printf("After function call: %d\n", x);   // 5 (unchanged!)
    
    return 0;
}

// Output:
// Before function call: 5
// Inside function: 15
// After function call: 5
Explanation:
  • The variable x has the value 5
  • When modifyValue(x) is called, a copy of 5 is passed
  • The function modifies the copy (5 + 10 = 15)
  • The original variable x remains unchanged (still 5)

Example: Multiple Parameters

float calculateArea(float length, float width) {
    return length * width;
}

int main() {
    float l = 5.5, w = 3.2;
    float area = calculateArea(l, w);  // Pass l and w as arguments
    printf("Area: %.2f\n", area);      // Output: Area: 17.60
    return 0;
}
Part 3

Variable Scope

Local and Global Variables

The scope of a variable determines where in the program it can be accessed. Variables can be either local or global.

Local Variables

Variables declared inside a function. They exist only within that function and cannot be accessed from outside.

Characteristics:
  • Declared inside a function or block
  • Only accessible within that function/block
  • Created when function is called
  • Destroyed when function exits
  • Each function call gets its own copy
void myFunction() {
    int x = 10;  // Local variable
    printf("%d", x);
}  // x is destroyed here

int main() {
    int y = 5;   // Local to main
    myFunction();
    // printf("%d", x); // ERROR! x doesn't exist here
    return 0;
}
Global Variables

Variables declared outside all functions. They can be accessed from any function in the program.

Characteristics:
  • Declared outside all functions (usually at top)
  • Accessible from any function in the file
  • Created when program starts
  • Destroyed when program ends
  • Automatically initialized to 0 if not explicitly initialized
int globalVar = 100;  // Global variable

void function1() {
    printf("Function1: %d\n", globalVar);
}

void function2() {
    globalVar = 200;  // Can modify global
    printf("Function2: %d\n", globalVar);
}

int main() {
    function1();  // Output: Function1: 100
    function2();  // Output: Function2: 200
    printf("Main: %d\n", globalVar);  // Output: Main: 200
    return 0;
}

Complete Example: Local vs Global

#include <stdio.h>

// Global variable
int globalCount = 0;

void increment() {
    int localCount = 0;  // Local variable
    
    localCount++;        // Increment local
    globalCount++;       // Increment global
    
    printf("Local: %d, Global: %d\n", localCount, globalCount);
}

int main() {
    increment();  // Output: Local: 1, Global: 1
    increment();  // Output: Local: 1, Global: 2
    increment();  // Output: Local: 1, Global: 3
    
    // localCount doesn't exist here
    printf("Final Global: %d\n", globalCount);  // Output: Final Global: 3
    
    return 0;
}
Explanation:
  • localCount is created fresh each time increment() is called, so it always starts at 0
  • globalCount persists across all function calls, so it accumulates: 1, 2, 3
  • localCount cannot be accessed from main()
⚠️ Best Practice:

Use local variables whenever possible. Global variables can make programs harder to debug and maintain because any function can modify them. Only use globals when you truly need to share data across many functions.

Key Takeaways

  • Modular programming breaks programs into independent functions
  • Divide and conquer: split complex problems into smaller, manageable sub-problems
  • Built-in functions: stdio.h (I/O), math.h (calculations), stdlib.h (utilities), time.h (date/time)
  • User-defined functions allow you to create custom functionality
  • Function header: return-type function-name(parameters) { body }
  • Void functions perform actions without returning values
  • Non-void functions return a value that can be used in expressions
  • Return statement sends a value back and exits the function
  • Arguments are passed by value (copy of data, not original)
  • Local variables: accessible only within their function, destroyed after execution
  • Global variables: accessible from any function, exist throughout program lifetime
  • Prefer local variables; use globals sparingly to avoid debugging complexity
